kubectl taint nodes <nodeName> key=value:taint-effect

NoSchedule
PreferNoSchedule
NoExecute

NoSchedule - Which means the pods will not be scheduled on the nodeName

PreferNoSchedule - Which means the system will try to avoid placing a pod on the node 
but that is not guaranteed 

NoExecute - New pods will not be scheduled on the node and existing pods
on the node if any will be evicted if they do not tolarate the taint

kubect taint nodes node1 app=myapp:NoSchedule

kubectl taint nodes desktop-worker function=webserver:NoSchedule


Remove taint, use - 

kubectl taint nodes desktop-worker function=webserver:NoSchedule-

Check current taint

kubectl describe node desktop-worker | grep -i taint

kubectl get node desktop-worker -o jsonpath='{.spec.taints}'

kubectl edit node desktop-worker



 kubectl describe node desktop-control-plane | grep Taint  (Linux)
